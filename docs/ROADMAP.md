# ShortFlow Studio Premiere 내부 베타 4주 로드맵

기준일: 2026-07-11  
목표: **한 달 안에 Premiere 내부 베타 완성**

현재 목표는 판매·상용화 버전이 아니라, Premiere 안에서 반복 편집 시간을 실제로 줄이는 내부 베타입니다. 상세 포함·제외 기준은 [내부 베타 범위](INTERNAL_BETA_SCOPE.md)를 단일 기준으로 사용합니다.

## 실행 원칙

- Premiere 실제 Host 검증은 Mock/정적 검증과 분리해 별도 게이트로 수행합니다. Mock 통과를 실제 Host 통과로 간주하지 않습니다.
- 현재 Premiere Pro 2026과 UXP Developer Tools에서 패널 로드, watch/reload, 빈 프로젝트 안전 처리, 테스트 MP4 import, 활성 시퀀스 기본 QC, Safe Zone overlay, SRT import, 음악/SFX WAV 기본 삽입, TrackItem 선택 감지, 자동 컷 dry-run·추천 마커·복제 시퀀스 적용까지 제한 통과했습니다. TTS live/API 삽입은 최종 승인 전 다시 확인합니다.
- Host 연동 파일은 고위험 영역이며, 실제 결과에 근거한 최소 수정만 허용합니다.
- 결제·라이선스·계정·서버·자동 텔레메트리·고급 AI·마케팅 기능은 구현하지 않고 후순위로만 기록합니다.
- 중간 커밋·push는 하지 않습니다. 전체 범위와 `typecheck`, `lint`, `test`, `build`가 통과한 뒤 체크포인트 커밋 하나만 만듭니다.
- GitHub push는 내부 베타 검증 완료 시점 또는 스프린트 종료 시점에 한 번만 합니다.

## 1주차 — 편집 기반과 자막·음성

순서를 변경하지 않습니다.

1. 요구사항·로드맵 문서 업데이트
2. 내부 베타 범위 확정
3. Mock Host 기반 Premiere bridge 안정화
4. 자막·STT·SRT 데이터 모델 안정화
5. 단어 타임스탬프 기반 자막 편집 안정화
6. TTS 생성·저장·삽입 기본 흐름 정리
7. Premiere 설치 후 실제 Host bridge smoke test

오늘의 완료 기준은 1~6번의 Mock/정적 근거입니다. 7번은 Premiere 설치 뒤 [Host Beta Runbook](HOST_BETA_RUNBOOK.md)에 따라 별도 수행합니다.

현재 1~6번, 음악/SFX 순서·미리듣기·폴더 카테고리, 에셋 권리 관리 코어, 최종 QC 권리 리포트, 썸네일 수동 편집·내보내기는 `npm run check` 1017/1017와 build/dist 검증으로 Mock 기준선을 확보했습니다. 7번 실제 Host smoke는 UXP 패널 로드, UDT watch/reload, 빈 프로젝트 안전 처리, 테스트 MP4 import, 활성 시퀀스 생성, 테스트 클립 삽입, 기본 QC, Safe Zone overlay, SRT import, 음악/SFX 삽입·소스 모니터 미리듣기, 플레이헤드·In/Out 상태 읽기, TrackItem 선택 감지, TTS/STT 탭 접근성, 자동 편집 입력 없음 안전 차단, 자동 컷 dry-run·추천 마커·복제 시퀀스 적용까지 제한 통과로 기록했습니다. TTS live/API 삽입은 내부 베타 최종 승인 전 재검증합니다.

## 2주차 — 자동 편집·오디오 자산·권리

1. 자동 컷·펀치인 기본 비파괴 기능
2. 음악·효과음 폴더 열기
3. 기존 효과음 파일 동기화
4. 폴더·카테고리 관리
5. 드래그 순서 이동
6. 미리듣기
7. 타임라인 삽입
8. 에셋 권리 관리 기본 구현
9. Premiere 실제 타임라인 삽입 검증

고급 비트 매칭과 자동 덕킹 고도화는 포함하지 않습니다.

현재 4~6번은 기본 필터·드래그 순서 저장·오디오 미리듣기 UI를 Mock 기준으로 보강했습니다. 7번은 실제 Host에서 `host-smoke-assets/SFX/shortflow_smoke.wav` 동기화, Premiere 프로젝트 import, A1 타임라인 삽입까지 기본 경로를 확인했습니다. 8번은 `src/asset-rights.ts`, 자산 패널 권리 입력 UI, 최종 QC 권리 리포트 연결까지 Mock 기준으로 추가했습니다. 실제 Host의 미리듣기·드래그 순서 이동·잠긴 트랙/충돌 경고는 최종 승인 전 추가 확인합니다.

## 3주차 — 썸네일·Safe Zone·로컬 안정성

1. 1280×720 Canvas 기반 수동 썸네일 편집
2. 1~4분할 배경
3. 이미지 이동·확대·크롭
4. 텍스트·배지·그림자·외부광선
5. 밝기·대비·채도
6. PNG/JPG 내보내기
7. Safe Zone 오버레이
8. 설정·자동 저장·복구·사용자 실행 진단 로그

썸네일 AI 대화 수정과 A/B 자동 판단은 포함하지 않습니다.

Premiere Pro 26.3 UXP Canvas는 `drawImage`, 텍스트 렌더링과 browser-style export API가 부족한 조합이 확인됐습니다. 내부 베타에서는 해당 환경에서 PNG/JPG 버튼을 비활성화하고, 원본 이미지/AI 결과 바이트를 data URL로 내장하는 SVG fallback 저장 버튼으로 최소 내보내기 경로를 제공합니다. Safe Zone 오버레이는 Canvas 없이 BMP 파일을 생성해 Premiere로 가져오는 경로로 전환했습니다. PNG/JPG Host 승인은 별도 렌더 경로가 확보된 뒤 다시 판정합니다.

- AI 레퍼런스 보드
- 외부에서 만든 이미지·영상 파일 가져오기

## 기존 1~13단계 흐름 보존

기존 1~13단계의 큰 흐름은 위 4주 계획으로 재배치하되 삭제하지 않습니다. Host/패키징/음성/자막/자동 편집/Safe Zone/오디오 자산/레퍼런스/썸네일/진단·복구/배포·권리/상용화 순서는 추적표에 유지합니다. 내부 베타에 불필요한 상용화 항목은 구현하지 않습니다.
